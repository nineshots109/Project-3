Project #2 - Authentication Portal Project

Overview

This project is designed to give students practical experience in building web applications with Flask, a Python web framework. The core functionality of the application will be user authentication, which includes a signup and login page. Once authenticated, users will be directed to a personalized portal that displays their information. All user data and additional required information will be stored in a local SQLite database. Students are encouraged to explore and utilize any libraries they find suitable for accomplishing the project goals.

Objectives

  • Understand the basics of web application development using Flask.
  • Implement user authentication (signup and login) in a web application.
  • Store and retrieve user data from a SQLite database.
  • Develop a user-specific portal that is accessible post-authentication.

Requirements

Environment Setup

  • Use Python 3.x for the project.
  • Install Flask and any other libraries you deem necessary.
  • Set up a local SQLite database for storing user information.

User Authentication

  • Implement a signup page where users can create a new account. The signup form should at least include username, email, and password fields.
  • Implement a login page where users can sign in using their credentials (username/email and password).
  • Implement session management to keep users logged in until they log out or the session expires.

Database Design

  • Design a SQLite database schema to store user information. At a minimum, the database should store usernames, passwords (hashed for security), emails, and any other information you require for the portal.
  • Include functionality for inserting new user data during signup and retrieving user data for authentication.

User Portal

  • Upon successful login, users should be redirected to their personalized portal.
  • The content of the portal is flexible and can be tailored to your interests. Examples include a dashboard displaying user data, a simple blog platform, a to-do list, or any other content you wish. However, the portal must be protected and only accessible to logged-in users.

Security Measures

  • Ensure that passwords are securely stored in the database (consider using hashing).
  • Optional: Implement necessary security measures to protect against common vulnerabilities such as SQL injection and Cross-Site Scripting (XSS).

Bonus Challenges

  • Implement password reset functionality.
  • Add the ability for users to update their profile information.
  • Implement an admin dashboard for user management (view, delete users).
